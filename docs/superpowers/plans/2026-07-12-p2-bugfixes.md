# P2 Bug Fixes Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix five bugs in already-shipped features: budgets that ignore month, wallet balances that never move, a personal task list with nowhere to live, journal auto-attendance that merges unrelated class periods, and a silently-unused recurring-task field.

**Architecture:** Five independent, single-concern changes across four files. Each is done as its own task with its own commit — none of the five depend on each other's code, only on the shared design doc's compatibility rules (budget legacy-doc fallback) and UI conventions (existing badge/list-item patterns already used elsewhere in the same files).

**Tech Stack:** Plain browser JS, no framework, no test runner (matches P0/P1 precedent in this repo). Verified with `node --check` per modified file, `grep` for structural confirmation, and a Playwright smoke test in the final task.

## Global Constraints

- Scope is exactly: `js/views/finance.js` (Tasks 1-2), `js/views/productivity.js` (Task 3), `js/views/teacher.js` (Task 4). `js/views/dashboard.js` is read-only reference (already correct, confirmed in the design doc — no changes needed there).
- No new dependencies, no schema migration scripts (none exist in this app — everything is client-driven Firestore reads/writes).
- Legacy `budgets` documents (no `bulan` field) must keep working as "applies to every month" until a specific month is explicitly (re)saved for that category — per the design doc's compatibility rule. Do not write a one-time migration.
- Wallet balance adjustments must not block saving the transaction itself if the wallet lookup/update fails (wrap in `try/catch`, transaction is always the source of truth).
- The existing "Tugas" tab (teacher-assigned, `class_tasks`) is untouched — its label and behavior stay exactly as-is. The new tab is purely additive.
